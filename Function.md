# Todo API 教學工具 - 功能說明文件 (Function.md)

這份文件記錄了本「Todo API 教學工具」中所有已實作的各項功能與對應的內部行為邏輯。由於這是一個純前端的網頁展示工具，所有的資料操作皆維護於瀏覽器記憶體中，並同步展示對應的「模擬 API 呼叫的狀態」。

## 1. 新增待辦事項 (Add Todo)
- **說明**：使用者可以在文字輸入框輸入待辦事項內容，並在旁邊的日期選擇器選擇所屬日期（預設自動帶入當天日期）。
- **流程**：
  1. 點擊「新增」按鈕後，會將資料加入到內部的 `todos` 陣列中。
  2. 每個新項目會被賦予一組獨一無二的 `id` (透過時間戳產生)、以及預設狀態 `completed: false`。
  3. **Mock API 行為**：會觸發狀態列出現 `POST /todos/` (HTTP 201 狀態碼)，並更新「請求紀錄」與中間的 JSON 回應區塊。

## 2. 刪除待辦事項 (Delete Todo)
- **說明**：在清單中的每一列待辦事項的右側，會有一個帶有紅色 X 圖示的「刪除」按鈕。
- **流程**：
  1. 點擊刪除按鈕時，內部會觸發 `deleteTodo(id)` 方法，從陣列中篩除對應 ID 的物件。
  2. **Mock API 行為**：會觸發狀態列出現 `DELETE /todos/{id}` (HTTP 200 狀態碼) 模擬刪除成功資訊。

## 3. 切換完成狀態 (Toggle Completion)
- **說明**：待辦事項清單前方有一個核取方塊（Checkbox），用來切換任務是否已完成。
- **流程**：
  1. 點擊核取方塊會觸發 `toggleTodo(id)` 將該項目的 `completed` 屬性在 `true` 與 `false` 間反轉。
  2. 介面上針對已完成的事項，文字會變成灰色並加上「刪除線」。
  3. **Mock API 行為**：會觸發狀態列出現 `PATCH /todos/{id}` (HTTP 200 狀態碼) 並回傳剛更新的狀態欄位。

## 4. 狀態分頁切換 (Tabs Filtering)
- **說明**：在「待辦事項」標題的下方有三個分頁籤：「全部」、「待完成」、「已完成」。
- **流程**：
  1. 系統自動隨時追蹤總事項數、未完成數量以及已完成數量，並更新在分頁籤上的數字括號內。
  2. 點擊任一分頁標籤，待辦事項面板會自動過濾並顯示符合「該狀態」的資料列。

## 5. 匯出紀錄格式 JSON (Export to store.json)
- **說明**：方便使用者將在頁面上測試的新增資料抓取下來保留。在左側卡片上方提供了一個「匯出 store.json」按鈕。
- **流程**：
  1. 點擊按鈕時，會將當前的狀態陣列 `todos` 透過 `JSON.stringify` 轉為 JSON 字串。
  2. 使用 Blob 以及動態建立的下載連結，直接呼叫瀏覽器層級的下載功能，自動將結果存儲成 `store.json`，整個過程不需要後端伺服器參與。

## 6. API 運作監控與連線動畫 (API Monitor)
- **說明**：這主要位於畫面的右半卡片，作為教學示意。顯示每次操作對於後端 API 可能產生的互動。
- **流程**：
  1. 包含網路連線節點圖的**閃爍小動畫**：當新增、刪除、切換時，連線會有光束閃過的視覺特效。
  2. **當前 Endpoint 顯示**：更新並標註 HTTP 開頭方法 (例如 POST/GET/DELETE) 會有對應的不同顏色標籤。
  3. **回應區塊即時著色**：將收到的假資料格式化成 JSON 結構並提供 Syntax Highlighting 色彩凸顯。
  4. **歷史紀錄列表**：自動由上往下堆疊所有的操作歷程與模擬花費時間（ms）。
